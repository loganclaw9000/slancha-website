Slancha Launch Playbook

Everything that needs to happen before, during, and after the API goes live.

Pre-Launch (T-14 days)

Waitlist Nurture Sequence
T-14 to T-7
  • Set up Loops.so account and connect domain for email delivery
  • Import existing waitlist emails from Supabase auth table
  • Create 4-email drip sequence:
    Email 1 (immediate): Welcome + "here's what Slancha does" + link to docs
    Email 2 (day 3): Case study / benchmark showing savings
    Email 3 (day 7): "Launch is coming" + early access offer
    Email 4 (launch day): "We're live — get your free API key"
  • Wire up Supabase signup → Loops contact sync (Edge Function or webhook)
  • Test the full sequence with a personal email
Content Staging
T-14 to T-3
  • Write launch blog post: "Introducing Slancha — World-class AI inference, no infrastructure team required"
  • Write Show HN post (draft, ~300 words): problem, what we built, how it works, link
  • Write Product Hunt listing: tagline, description, 4 screenshots, maker comment
  • Write Twitter/X launch thread (8-10 tweets): hook → problem → solution → demo GIF → benchmarks → pricing → CTA
  • Write r/LocalLLaMA post: technical deep dive on routing + fine-tuning approach
  • Write r/MachineLearning post: research angle on task-specific models vs frontier
  • Record 60-second demo video (Loom or similar) showing: signup → get key → make request → see routing
  • Create demo GIF for Twitter/GitHub: terminal showing request + response with routing info
Technical Readiness
T-7 to T-1
  • Backend API live at api.slancha.ai with health check endpoint
  • Verify OpenAI SDK compatibility: openai.OpenAI(base_url="https://api.slancha.ai/v1")
  • Load test: 100 concurrent requests, p99 latency < 200ms (excluding model inference)
  • Verify Supabase auth → API key creation → first request flow end-to-end
  • Verify Stripe checkout works for Hobbyist and Pro tiers
  • Status page at /status shows all systems green
  • Monitoring/alerting set up (uptime, error rate, latency)
  • Rate limiting configured for free tier (10K/month)
  • Update site: change all "Join Waitlist" CTAs to "Get Started Free"
  • Remove any "coming soon" or "preview" language from the site

Launch Day (T-0)

Morning (6am-9am)
T-0 Morning
  • Final smoke test: signup → login → create key → make API request → see dashboard
  • Deploy any last site updates, verify GitHub Pages build succeeds
  • Send Email 4 (launch email) to entire waitlist via Loops
  • Publish launch blog post on the site
Launch Posts (9am-11am ET)
T-0 Mid-morning
  • Hacker News — Post "Show HN: Slancha — OpenAI-compatible API that auto-routes and fine-tunes your models"
    Timing: 9-10am ET for best visibility
  • Product Hunt — Launch listing goes live (schedule the night before)
  • Twitter/X — Post the launch thread from @slancha_ai account
  • Reddit — Post to r/LocalLLaMA (technical), r/MachineLearning (research angle)
  • LinkedIn — Founders post personal launch announcements
  • Dev.to / Hashnode — Cross-post the launch blog
All Day — Engage
T-0 All day
  • Monitor HN comments — respond to every question within 30 minutes
  • Monitor Product Hunt comments — respond to all
  • Monitor Reddit threads — be helpful, technical, transparent
  • Monitor Twitter mentions — retweet, reply, thank
  • Monitor signups in real-time (Supabase dashboard + Loops)
  • Monitor API health: error rate, latency, throughput
  • Fix any bugs immediately — fast iteration shows credibility
  • If HN hits front page: be ready for 10K+ signups in hours

Post-Launch (T+1 to T+14)

Week 1: Convert & Learn
T+1 to T+7
  • Email new signups who haven't made an API request yet — "Need help getting started?"
  • Identify top 10 most active free users → personal email offering Pro upgrade
  • Publish "Launch retrospective" blog post with real numbers (signups, requests, feedback)
  • Collect and publish testimonials/tweets from early users
  • Start tracking: signup → first request → 10th request → paid conversion funnel
  • Set up weekly email digest to waitlist/users: what's new, what we shipped
Week 2: Scale & Iterate
T+8 to T+14
  • Publish first real case study from a launch user
  • Submit to AI tool directories: There's An AI For That, AI Tool Finder, etc.
  • Submit MCP server to Official MCP Registry, Smithery, Glama
  • Start outbound to 50 companies spending $5K+/month on LLM APIs
  • A/B test pricing page (free tier CTA copy, pricing anchors)
  • Publish comparison pages: "Slancha vs OpenRouter", "Slancha vs Fireworks", etc.

Success Metrics

Track these from day one:

Signups: 500+ week 1 First API call: 30%+ of signups 10+ requests: 15%+ of signups Pro conversion: 2-3% of signups HN upvotes: 100+ PH upvotes: 200+ Twitter impressions: 50K+ Time to first request: < 5 min

Draft Launch Content

Show HN Post (draft)

Show HN: Slancha — OpenAI-compatible API that auto-routes and fine-tunes your models

We built Slancha because world-class AI inference shouldn't require a world-class infrastructure team.

It's a single API endpoint (OpenAI-compatible — swap your base_url, done). Behind it, Slancha automatically:

  • Routes each request to the optimal model (semantic classification, sub-ms overhead)
  • Analyzes your usage patterns to curate training data
  • Fine-tunes smaller, task-specific models that match frontier quality at 40-60% less cost
  • Optimizes inference with quantization and GPU packing
  • Redeploys improved models continuously

The longer you use it, the cheaper and better it gets — your fine-tuned 7B can outperform GPT-4 on your specific workloads.

Free tier: 10K requests/month (hard cap, no overage), 20+ models, BYOK — your API key pays the inference provider directly. No credit card.
Hobbyist $9/mo (50K req/mo, then $0.50/1K), Pro $249/mo (100K req/mo, then $0.40/1K, plus 2 LoRA adapters + 1 training run/mo), Enterprise $2K+/mo with volume pricing. Half of Not Diamond's $1.00/1K on Hobbyist; 60% less on Pro.

Docs: slancha.ai/docs | Signup: slancha.ai/signup

We're two founders — happy to answer any questions about the routing architecture, fine-tuning approach, or business model.
